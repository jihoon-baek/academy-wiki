기초 원칙
원칙1: /instructions 폴더의 모든 md파일을 읽어 현재 만들고자 하는 웹사이트에 대해 명확하게 이해해야 한다.
원칙2: AI의 경우 사용자가 '코딩해줘'라고 명확하게 코드 수행을 명령하지 않는 이상 코드 작성 및 수정을 할 수 없다.
원칙3: 위의 원칙과 더불어 모든 유저의 질문/프롬프트에 대해 AI는 기본적으로 '질문 이해, 코드 분석, 상황 이해, 방향성/코드수정방향성 등 설계, 답변'만을 진행하는 것을 대원칙으로 한다.
원칙4: 위의 원칙2와 원칙3에 따라 AI는 사용자가 '코드 수정해줘'라고 명령할 경우에만 코드 수정을 할 수 있다.
원칙5: 모든 코딩 수행 후에는 Github 업로드를 수행하는 것을 대원칙으로 한다.
원칙6: Github 업로드 후에는 반드시 Github에 대한 설명을 작성하는 것을 대원칙으로 한다.
원칙7: Github 업로드 직후의 답변에서는 반드시 이번 Github 업로드에 대한 설명을 포함해야 한다.(번호, 파일명, 변경사항, 변경된 파일의 주요 내용 등)

### 기존 문제점 / 개선의 이유
기본 설정이 `/w/` 하위 경로로 되어 있어 `localhost:8080` 메인 페이지의 설치 링크가 404 에러를 발생시켰음.
### 개선된 부분
1. `docker-compose.yml`의 모든 경로를 `/var/www/html/w`에서 `/var/www/html`로 변경하여 루트 기반으로 작동하게 수정.
2. Apache 설정 파일(`apache.conf`)을 별도로 생성하여 컨테이너에 마운트함. 이를 통해 `/w/` 없이 루트에서 바로 PHP와 RewriteRule이 작동하도록 개선.
3. `MW_SCRIPT_PATH`를 빈 값(`''`)으로 설정하여 미디어위키 내부 링크가 루트 주소로 생성되도록 동기화.
### 이로 인해 개선되어야 하는 부분
추가적인 주소 입력 없이 `http://localhost:8080`에 접속하면 바로 설치 가이드가 나타나며, 모든 이미지와 스크립트가 정상적으로 로드됨.
